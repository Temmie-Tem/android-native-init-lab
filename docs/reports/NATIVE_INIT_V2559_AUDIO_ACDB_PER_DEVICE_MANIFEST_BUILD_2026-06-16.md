# V2559 — Audio ACDB per-device manifest capture build

Date: 2026-06-16

## Decision

`v2559-acdb-per-device-manifest-capture-host-only-ready`

V2558 closed the reconciliation gap: the common topology payload is pinned by SHA-256, but full native ACDB replay remains blocked because the per-device AFE/ASM/ADM/AUDPROC/VOL payloads are not captured.  V2559 builds the next capture helper without touching a device: initialize ACDB, arm the `acdb_ioctl` dump preload, skip the already-pinned common topology call, then call `acdb_loader_send_audio_cal_v5(15, 0, 0x11135, 48000, 48000, 0, 1)` to expose the per-device GET out-buffers.

## Scope

- Host-only build and static validation.
- No Android boot, native flash, playback, mixer write, or `/dev/msm_audio_cal` open.
- No raw ACDB bytes committed.
- Private ARM32 artifacts live under `workspace/private/builds/audio/v2559-acdb-per-device-manifest-capture-host-only/`.

| Pinned topology SHA-256 | `7c5d45efa40944bc23dcc83af9f0046249499bb13d1a03c3470c287127992b89` |
| Per-device call | `acdb_loader_send_audio_cal_v5(15, 0, 0x11135, 48000, 48000, 0, 1)` |
| Preload policy | post-init auto-arm/manual-arm, no exit on 4916, fake audio-cal ioctls under `A90_ACDB_FAKE_ALLOCATE=1` |

The helper intentionally does not reference `acdb_loader_send_common_custom_topology` and does not issue calibration-device calls.  The combined preload remains responsible for faking `AUDIO_ALLOCATE_CALIBRATION`, `AUDIO_DEALLOCATE_CALIBRATION`, and `AUDIO_SET_CALIBRATION`; live use must keep real SET pass-through at zero.

## Live Boundary

This is not a live capture.  A future live unit may stage these private artifacts through the existing V2490/V2555 Android own-process handoff pattern, with the existing fake-allocate/fake-deallocate/fake-set measurement envelope.  Acceptance for that future unit should require at least one `ret==0`, non-all-zero ACDB out-buffer whose length is not `4` or `4916`, plus zero real `AUDIO_SET_CALIBRATION` pass-through events.

If the helper returns no per-device records, the next branch is not native replay; it is a capture-path diagnosis of whether `send_audio_cal_v5` requires prior common-topology side effects or different argument geometry.

Native full-calibration replay remains blocked until the per-device payload manifest is explicit: command/order, lengths, SHA-256, memory-handle policy, cleanup, and rollback behavior.
