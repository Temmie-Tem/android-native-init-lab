# V2561 — Audio ACDB topology-skip per-device build

Date: 2026-06-16

## Decision

`v2561-acdb-toposkip-per-device-capture-host-only-ready`

V2560 proved the previous helper never reached `send_audio_cal_v5`: `acdb_loader_init_v3` itself entered the common-topology tail and segfaulted before returning.  V2561 builds the next host-only capture artifact: a combined preload that still dumps `acdb_ioctl` and fakes audio-cal ioctls, but also exports a no-op `acdb_loader_send_common_custom_topology()` that returns success before the crashing real tail.

## Scope

- Host-only build and static validation.
- No Android boot, native flash, playback, mixer write, or `/dev/msm_audio_cal` open.
- No raw ACDB bytes committed.
- Private ARM32 artifacts live under `workspace/private/builds/audio/v2561-acdb-toposkip-per-device-capture-host-only/`.

## Design Basis

Host `readelf` on the stock `libacdbloader.so` shows `acdb_loader_send_common_custom_topology` is both exported and reached through an `R_ARM_JUMP_SLOT` relocation.  That makes a preload-level symbol interposer a plausible host-only next step.

The V2561 preload exports:

- `acdb_ioctl` — existing ACDB out-buffer dumper.
- `ioctl` — existing fake `AUDIO_ALLOCATE`/`DEALLOCATE`/`SET` measurement gate.
- `a90_arm_capture` — existing capture arm hook.
- `acdb_loader_send_common_custom_topology` — new short-circuit returning `0` and logging a private marker.

The topology payload is not discarded as evidence; it remains pinned from V2548/V2557:

- Length: `4916`.
- SHA-256: `7c5d45efa40944bc23dcc83af9f0046249499bb13d1a03c3470c287127992b89`.

## Static Checks

- helper is a 32-bit ARM PIE with `/system/bin/linker` interpreter.
- helper has undefined `acdb_loader_init_v3`, `acdb_loader_send_audio_cal_v5`, and weak `a90_arm_capture`.
- preload exports `acdb_ioctl`, `ioctl`, `a90_arm_capture`, and `acdb_loader_send_common_custom_topology`.
- topology-skip source does not call `dlsym` or `RTLD_NEXT`; it does not call the real common-topology function.
- no live device action, speaker write, or calibration-device call is added by this build unit.

## Live Boundary

This is not a live capture.  A future V2562 live unit can stage the V2561 preload through the same V2490/V2560 Android handoff path and should accept success only if:

1. a topology-skip marker appears;
2. the helper reaches `before_send_audio_cal_v5` or `send_audio_cal_v5_return`;
3. at least one `ret==0`, non-all-zero ACDB out-buffer appears with `out_len` not in `{4, 4916}` and not from the known init/topology command set; and
4. real `AUDIO_SET_CALIBRATION` pass-through remains `0`.

If V2562 still fails before `send_audio_cal_v5`, the next branch should be direct per-device GET command RE rather than another common-topology skip variant.
