# v346 Plan: Isolate V317 Handoff Preflight Output

- date: `2026-05-19`
- scope: host-side V317 handoff command safety
- boot image change: none planned
- device mutation: none planned
- status: implemented / pending post-commit refresh

## Summary

The V340 handoff packet prints a preflight command before the live `run` command.
Before v346, that preflight command was generated by changing only the final
subcommand from `run` to `preflight`, so it kept the live proof output directory:

```text
tmp/wifi/v317-private-property-namespace-proof
```

That is unsafe operationally because a successful no-device preflight could write
a `private-property-namespace-proof-preflight-ready` manifest into the same path
that V333 uses as the live V317 result. v346 changes the handoff packet so the
preflight command writes to a dedicated evidence directory.

## Implementation

- Update `scripts/revalidation/wifi_v317_handoff_packet.py`.
- Add dedicated preflight output directory:
  - `tmp/wifi/v317-private-property-namespace-proof-preflight`
- Generate `preflight_command` with:
  - final subcommand `preflight`
  - unchanged exact V317 approval phrase
  - unchanged `--prelive-gate-manifest`
  - isolated `--out-dir`
- Add handoff checks:
  - `preflight-command-contract`
  - `preflight-output-isolated`
